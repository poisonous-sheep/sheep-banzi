# 强连通分量 (SCC) 与 缩点 🚀

这是一个基于 **Tarjan 算法** 的强连通分量（Strongly Connected Components, SCC）求解与缩点重构图的算法讲解文档。常用于将复杂的有环有向图转化为 **有向无环图 (DAG)**，进而在 DAG 上利用拓扑序进行动态规划或连通性分析。

## 🧩 例题一：P3387 【模板】缩点

### 1. 问题抽象

给定一个带点权的有向图，图中可能包含环。你需要找到一条路径，使得路径经过的点权之和最大。**注意：一个点如果被经过多次，其点权只计算一次。**

### 2. 核心思想 (Tarjan 与 缩点)

在有向图中，如果存在一个环，那么环上的任意两个点都可以互相到达。由于题目允许走回头路且点权只计算一次，**只要我们走到了环上的任意一点，就必然能且一定会把整个环上所有点的点权都收集到**。

因此，我们可以把图中所有能互相到达的点集（即强连通分量 SCC）**“压缩”成一个超级节点**（缩点）。这个超级节点的权重就是原环上所有点权之和。缩点后，原图中的环被消除，图退化为 **有向无环图 (DAG)**，我们就可以在 DAG 上进行动态规划。

### 3. 算法流程与原理剖析

#### Tarjan 如何求 SCC

Tarjan 算法的核心是 DFS（深度优先搜索）和栈。它通过维护两个时间戳数组来判断环：

- **`dfn[u]` (发现时间)**：DFS 遍历到节点 `u` 的次序（绝对递增的时间戳）。
    
- **`low[u]` (追溯值)**：从 `u` 或 `u` 的子树出发，通过**最多一条非树边**（即返祖边/横叉边）能够到达的、在栈中最老的节点的时间戳。
    

#### 为什么 `low[u] == dfn[u]` 就是一个 SCC？

在 DFS 过程中，如果一个节点 `u` 的所有子树都遍历完了，且它的 `low[u]` 依然等于 `dfn[u]`，说明**它的子树中没有任何一条边能够指向上方（祖先节点）**。

这意味着 `u` 就是当前这个强连通分量在 DFS 树中的**最高点（根节点）**。此时，栈中从栈顶到 `u` 的所有节点，构成了一个完整的、无法再向上扩展的 SCC。我们将其不断出栈，直到 `u` 被弹出。

#### `bel[]` 和 `sz[]` 的作用

- **`bel[u]` (Belong)**：记录原图中的节点 `u` 属于缩点后的哪一个新 SCC（超级节点）的编号。
    
- **`sz[i]` (Size/Sum)**：记录第 `i` 个 SCC 的总点权（或包含的原节点个数）。
    

#### 缩点后为什么是 DAG？

由于我们把所有存在环的结构都合并成了一个个独立的超级节点，剩余的边必定都是连接两个**不同 SCC** 的单向边。如果两个 SCC 之间还能形成环，它们在 Tarjan 过程中早就会被合并成同一个 SCC。因此，缩点后的图绝对没有环，必定是 DAG。

### 4. DAG 上的动态规划

在这个绝对无环的图上，求最大权值路径就是一个经典的 DAG 上 DP 问题。

#### Trick：为什么可以直接按编号倒序 DP？这里为什么没写拓扑排序？

仔细观察代码：

```
lep(ct, 1) { // 从总 SCC 数量倒序遍历
    for (auto v : edge[i]) dp[v] = max(dp[v], dp[i] + sz[v]);
}
```

**不需要写拓扑排序，这是 Tarjan 算法的一个极其优雅的隐藏性质！**

Tarjan 在找 SCC 时，最先找到（出栈）的 SCC，一定是 DFS 树底部的“叶子” SCC（即出度为 0，或只能指向已出栈 SCC 的部分）。

因此，**SCC 的编号 `ct` 实际上是天然的「逆拓扑序」**。编号 1 的 SCC 是图的汇点（Sink），编号 `ct` 的 SCC 通常是图的源点（Source）。

我们直接从 `ct` 循环到 1，就等同于**完美的拓扑排序正向遍历**。这省去了使用队列和入度数组跑 Kahn 算法的大量代码。

#### 多重边是否影响答案？

缩点建图时：

```
if (bel[i] != bel[v]) edge[bel[i]].push_back(bel[v]);
```

这里没有去重，所以两个 SCC 之间可能会有多条相同的边。对于求**最大路径和**来说，同一条边走多次和走一次 `max` 的结果是一样的，因此完全**不影响正确性**。但如果是求“最短路计数”或“路径方案数”，则必须去重！

## 🧩 例题二：P2341 受欢迎的牛

### 1. 问题抽象 (图论建模)

给定 $N$ 头牛和 $M$ 对“喜欢”关系，A 喜欢 B 是一条 $A \to B$ 的有向边。喜欢关系具有传递性。

**目标**：寻找有多少头牛被**所有其他牛**喜欢。转化为图论语言即：**寻找图中有多少个节点，能被其他所有节点到达**。

### 2. 核心思想与缩点必要性

#### 为什么不能直接在原图上做？

如果原图有环（互相喜欢的牛群），环里的牛性质完全相同：只要有一头牛被大家喜欢，环里其他牛也一定被大家喜欢。如果在原图上直接找，环内复杂的连通关系会破坏判定条件（无法简单用出入度衡量）。

#### 为什么要缩点？

缩点后，图变成了 DAG。在 DAG 中讨论可达性就极其简单了：

被所有人到达的点，**必须是图的唯一终点（汇点）**。

### 3. `to[]` 数组与出度判断

#### `to[]` 数组表示什么？

`to[i]` 表示第 `i` 个 SCC 的**出度**。

代码中：`if (bel[i] != bel[v]) to[bel[i]]++;` 统计了每个超级节点向外指出的边数。

#### 为什么只关心 出度为 0 的 SCC？

在 DAG 中，“所有点都能到达它” 等价于 “它是全图唯一的终点”。

- 如果一个 SCC 的出度大于 0，说明它还会通向别人。顺着有向边，它要去“喜欢”别人，但别人无法“喜欢”回来（因为没有环）。所以它绝对不是全图最受欢迎的。
    

#### 关键结论推导 (反证法)

- **如果出度为 0 的 SCC** $\ge 2$ **个** $\to$ **答案是 0**。
    
    反证：假设有两个终点 A 和 B。既然它们出度都为 0，意味着 A 到不了 B，B 也到不了 A。全图不可能存在任何一个点能被“所有人”到达。
    
- **如果出度为 0 的 SCC 只有 1 个** $\to$ **答案是它的大小**。
    
    因为图是连通的（如果不连通必定会产生多个出度为 0 的点），且 DAG 所有的路径最终必定流向这唯一的一个汇点。所以这个 SCC 里的所有牛就是我们要找的“受欢迎的牛”，输出 `sz[i]`。
    

## ⚡ Trick / 易错点总结

在实际手写 Tarjan 和缩点 DP 时，极其容易踩入以下陷阱，请务必结合代码反复核对：

### 1. 🚨 Tarjan `low` 数组的更新失误

```
if (!dfn[v]) dfs(dfs, v);
if (ins[v]) low[u] = min(low[u], low[v]); // 这里极其容易错！
```

**易错点**：如果少了 `if (ins[v])` 的判断，会把**已经出栈的**（属于其他 SCC 的）点的 `low` 值混入当前节点。返祖边必须指向**还在当前搜索栈中**的节点！写错会导致 SCC 划分完全混乱。

### 2. 🚨 重建图时忘记判环

```
if (bel[i] != bel[v]) edge[bel[i]].push_back(bel[v]); 
```

**易错点**：绝不能直接写 `edge[i].push_back(v)`。必须判断 `bel[i] != bel[v]`。如果不加这个判断，会把同一个 SCC 内部的边加到新图中形成自环。虽然求最大权可能不影响，但在跑度数统计（如 P2341 的 `to[]`）时会导致**出度计算错误**！

### 3. 🚨 DAG DP 的隐藏遍历方向

```
lep(ct, 1) // 从 ct 到 1 倒序递减
```

**易错点**：如果你习惯性地写成了正向循环 `rep(1, ct)`，DP 答案将完全错误！因为 `ct` 的生成是逆向的，编号越大的 SCC 越靠“前”（入度为0的方向）。必须**倒序遍历**才能保证依赖的前置状态已经被计算完毕。

### 4. 🚨 孤立点与不连通图的边界

**本题（P2341）结论完美兼容图不连通的情况。**

如果图分为两个完全不连通的部分，那么每个部分至少会产生一个出度为 0 的 SCC。全图总共就有 $\ge 2$ 个出度为 0 的点，代码自然会输出 `0`，无需额外写并查集或 BFS 去特判图的连通性。但如果你在其他图论题里，不要忘记 `rep(1, n) if(!dfn[i]) dfs(i);` 以防漏掉不连通的子图。

### 5. 🚨 DP 数组的初始化

```
rep(1, ct) dp[i] = sz[i];
```

**易错点**：在 DAG 上求最大权值路径，每个点自身的基础权值 `sz[i]` 必须作为 DP 数组的初值。因为一条合法路径最少也会包含当前这一个超级节点，如果初始化为 0，会漏掉起点本身的权值。

## 💻 核心代码模板 (Tarjan SCC + 缩点 + DAG DP)

> _基于 P3387 【模板】缩点，使用原生代码，增加核心注释_

```
#include <bits/stdc++.h>
using namespace std;

#define ll long long
#define endl '\n'
#define rep(l, r) for (int i = l; i <= r; i++)
#define lep(r, l) for (int i = r; i >= l; i--)
#define int long long

void solve() {
    int n, m;
    cin >> n >> m;
    vector<vector<int>> adj(n + 1);
    vector<int> a(n + 1);
    rep(1, n) cin >> a[i];
    rep(1, m) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }
    
    // --- 1. Tarjan 数据结构初始化 ---
    stack<int> stk;
    // dfn: 发现时间, low: 追溯时间, bel: 缩点后的所属编号, ins: 是否在栈中, sz: 缩点后的点权和
    vector<int> dfn(n + 1, 0), low(n + 1, 0), bel(n + 1, 0), ins(n + 1, 0), sz(n + 1, 0);
    int tot = 0; // 时间戳
    int ct = 0;  // 强连通分量 (SCC) 的数量
    
    // --- 2. Tarjan 核心逻辑 ---
    auto dfs = [&](auto&& dfs, int u) -> void {
        dfn[u] = low[u] = ++tot;
        ins[u] = 1;
        stk.push(u);
        
        for (auto v : adj[u]) {
            if (!dfn[v]) dfs(dfs, v); // 树枝边：未访问过，向下深搜
            // 【易错点】必须判断 v 是否还在栈中 (ins[v])，才能更新 low
            if (ins[v]) low[u] = min(low[u], low[v]); 
        }
        
        // 发现一个强连通分量的根
        if (low[u] == dfn[u]) {
            ct++;
            while (1) {
                int v = stk.top();
                stk.pop();
                sz[ct] += a[v]; // 累加点权
                ins[v] = 0;     // 出栈标记
                bel[v] = ct;    // 记录归属
                if (v == u) break;
            }
        }
    };
    
    // 遍历所有点，处理不连通图的情况
    rep(1, n) if (!dfn[i]) dfs(dfs, i);
    
    // --- 3. 缩点建新图 (DAG) ---
    vector<int> ro(ct + 1, 0);
    vector<vector<int>> edge(ct+1);
    rep(1, n) {
        for (auto v : adj[i]) {
            if (bel[i] != bel[v]) { // 【易错点】去自环：不在同一个 SCC 里的边才加
                edge[bel[i]].push_back(bel[v]);
            }
        }
    }
    
    // --- 4. DAG 上跑 DP ---
    vector<int> dp(ct + 1, 0);
    rep(1, ct) dp[i] = sz[i]; // 【易错点】初始化必须为该超级节点自身的权值
    
    // 【Trick】Tarjan 产生的编号本身就是逆拓扑序，直接倒序遍历即可完美等价于拓扑排序
    lep(ct, 1) {
        for (auto v : edge[i]) {
            dp[v] = max(dp[v], dp[i] + sz[v]); // 状态转移
        }
    }
    
    // --- 5. 统计答案 ---
    int ans = 0;
    rep(1, ct) ans = max(ans, dp[i]);
    cout << ans << endl;
}

signed main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    int t = 1;
    // cin >> t;
    while (t--) solve();
    return 0;
}
```
